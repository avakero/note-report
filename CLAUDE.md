# CLAUDE.md

運用方針・配布チャンネル構成・リモート運用ファイルの説明はすべて **README.md** にある。作業前に必ず読むこと。

## このリポジトリの性質

- GitHub Pages 配信のブックマークレット配布リポジトリ。**mainへのマージ＝全ユーザーへの即時リリース**（ユーザーのブックマークは毎回 `note-tool.js` を読み直す）
- **公開リポジトリ**。コミットメッセージ・ドキュメントにも書ける内容か常に意識する（配布URLの秘匿は弱い前提）
- ユーザーはIT初心者層。レポートやページの文言は「やさしい日本語＋あばけろ君🐸のトーン」を守る

## 変更時の必須事項

- `note-tool.js` を変えたら: `node --check note-tool.js` と `node dev/test-build.js && node dev/test-analyze.js` を通す
- チャンネル出し分け（`liteMode`）・スナップショット仕様（1日1件・400日保持・明細は直近30件）を変えるときは README の「データの性質」も更新する
- 設置ページのブックマークレットを触るときは **PC用ボタン・スマホ用textarea・コンソール用textareaの3箇所**が同じコードになっているか確認（dev/test にデコード検証あり）
- 新機能はまず「わくわく版のみ」か「全チャンネル」かを確認する。コラボ基本版（`__NOTE_CHANNEL='collab'` かつ `__NOTE_PLUS` なし）には新機能セクションを出さない方針
