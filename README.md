# note わくわく分析レポート — 運用ガイド

note.com の自分のデータを分析して1枚のHTMLレポートにするブックマークレット型ツールの配布リポジトリ。
GitHub Pages（https://avakero.github.io/note-report/）でそのまま配信される。

## 仕組みの全体像

- ユーザーが持つのは数行の**ローダー（ブックマーク）**だけ。押すたびにこのリポジトリの `note-tool.js` を読み込むため、**mainにマージした変更は全ユーザーに即反映**される（ブックマークの配り直し不要）。
- ローダーには「名札」（`__NOTE_CHANNEL` / `__NOTE_PLUS`）が入っていて、ツール本体がそれを見て挙動を変える。

## 配布チャンネル一覧

| 設置ページ | 名札 | レポート内容 | デザイン |
|---|---|---|---|
| `install-wakuwaku-k7f2.html` | なし | 全機能 | あばけろ君（標準） |
| `install-collab-r8k3.html` | `__NOTE_CHANNEL="collab"` | 基本版（「前回とくらべて」「日別スキの推移」なし） | まきまきコラボ |
| `install-collab-plus-w6b3.html` | `__NOTE_CHANNEL="collab"` + `__NOTE_PLUS=true` | 全機能（ブックマーク名は「note分析＋」） | まきまきコラボ |

- 設置ページはすべて `noindex`。URLを知っている人にだけ案内する運用。
- 出し分けの実装は `note-tool.js` の `liteMode`（collabかつPLUSなし→新機能セクション非表示）。

## マージ不要で運用できるファイル（編集して置くだけで即反映）

| ファイル | 役割 |
|---|---|
| `notice.json` | 全ユーザー向けお知らせ枠（レポート上部）。`enabled: true` で表示 |
| `notice-<チャンネル>.json` | チャンネル専用お知らせ。存在すればこちらが優先、なければ `notice.json` に落ちる |
| `brand-<チャンネル>.json` | チャンネル専用のレポートデザイン（`hero`: 画像ファイル名 / `credit`: フッターのクレジット行）。ないチャンネルは標準デザイン |
| `hero-collab.jpg` | まきまきコラボのヒーロー画像（正方形推奨） |

## 新しいコラボを増やす手順（コード変更不要）

1. `install-collab-r8k3.html` を複製し、ローダー内の `__NOTE_CHANNEL` を新しい名前（例 `"collab2"`）に変える。ファイル名は `install-collab2-XXXX.html`（XXXXはランダム4文字）
2. `brand-collab2.json` を `brand-collab.json` と同じ形式で作る
3. コラボ画像（正方形1枚）をリポジトリに置く

機能追加版も出す場合は、複製したページのローダーに `window.__NOTE_PLUS = true;` を1行足した設置ページをもう1枚作る（PC用ボタン・スマホ用コード・コンソール用コードの3箇所すべて）。

## コラボの機能追加版を告知するとき

1. 機能追加版の設置ページURL（w6b3）を対象者に案内する
2. `notice-collab.json` を `enabled: true` にして告知文を入れる
3. `faq-collab.html` に新機能のQ&Aを戻す（git履歴の「新機能の告知をお知らせ枠とFAQに追加」コミットに完成形がある）

## データの性質（サポート時に重要）

- 「前回とくらべて」の記録は**ユーザーのブラウザ内**（note.comのlocalStorage、キーは `noteReportSnaps_ユーザー名`）。外部送信はしない
- 1日1件（同日は上書き）、最大400日保持。記事ごとの明細は直近30件のみ、古い分は合計だけにスリム化
- **別ブラウザ・別端末には引き継がれない**。ブラウザの閲覧データ削除で消える。シークレットモードでは保存できない
- 基本版でも記録は裏で動いているので、機能追加版に乗り換えた時点で過去分からグラフが出る

## 注意事項

- note の非公式APIを利用しているため、noteの仕様変更で突然動かなくなる可能性がある。安全装置（10分クールダウン・自動ディレイ・リクエスト上限・混雑時バックオフ）は `note-tool.js` に実装済み
- このリポジトリは公開のため、ファイル一覧（配布ページのURL含む）は誰でも閲覧できる。URLの秘匿はあくまで「案内した人以外は気づきにくい」程度の性質

## 開発

- 自動テスト: `node dev/test-build.js && node dev/test-analyze.js`（Node単体で動く。note APIはモック）
- レポートの見た目確認: テストが `dev/report-case2.html` を出力するのでブラウザで開く
